Offensive Security Lab & Initial Access

Overview

Established a dedicated, isolated research environment to simulate real-world cyberattacks and defense strategies. The primary goal of this project was to validate a secure "Attacker vs. Target" workflow and successfully execute a known exploit against a vulnerable Linux service.

The Architecture

The lab is built on VMware Workstation Pro 25H2, utilizing a strictly isolated virtual network to ensure no traffic escapes into the local host network.

  • Attacker Node: Kali Linux (Rolling Edition)
  • Target Node: Metasploitable2
  • Networking: Host-Only Adapter (Internal Sandbox)

Methodology & Execution

The exploitation followed the standard PTES methodology:

  • Reconnaissance: Stealthy SYN scan using Nmap to identify active services.
  • Vulnerability Analysis: Cross-referenced services (VSFTPD/Bind) against Exploit-DB.
  • Exploitation: Leveraged Metasploit Framework (MSF) to deliver payloads.
  • Post-Exploitation: Established a reverse shell session to verify system-level access.

Key Takeaways

  • Environment Parity: Hypervisor isolation is critical for safe testing.
  • Service Mapping: Identifying specific versions is 90% of the work.
